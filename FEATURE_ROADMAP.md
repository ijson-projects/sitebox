## 🚀 优先级改进建议

### 🔥 高优先级（核心功能增强）

#### 1. **多应用管理** ⭐⭐⭐⭐⭐
**为什么重要**：用户通常需要包装多个网站，这是竞品的核心优势
- 支持创建多个 Web App 实例
- 应用列表管理界面
- 快速切换不同应用
- 每个应用独立配置（URL、图标、窗口大小等）

**实现建议**：
```swift
// 应用配置模型
struct WebAppConfig: Codable, Identifiable {
    let id: UUID
    var name: String
    var url: String
    var iconPath: String?
    var windowWidth: Double
    var windowHeight: Double
    var createdAt: Date
}

// 应用管理器
class WebAppManager: ObservableObject {
    @Published var apps: [WebAppConfig] = []
    @Published var currentApp: WebAppConfig?
    
    func createApp(config: WebAppConfig) { }
    func deleteApp(_ app: WebAppConfig) { }
    func switchApp(_ app: WebAppConfig) { }
}
```

#### 2. **暗色模式支持** ⭐⭐⭐⭐⭐
**为什么重要**：macOS 用户强烈需求，提升用户体验
- 自动跟随系统暗色模式
- 手动切换明暗主题
- 支持自定义主题颜色

**实现建议**：
- 使用 `@Environment(\.colorScheme)` 检测系统主题
- 为 WebView 注入 CSS 来适配网站暗色模式
- 设置界面添加主题选择器

#### 3. **用户脚本注入（User Scripts）** ⭐⭐⭐⭐
**为什么重要**：允许用户自定义网站行为，增强功能
- 支持 JavaScript 脚本注入
- 脚本管理器（添加、编辑、删除）
- 脚本执行时机控制（页面加载前/后）
- 常用脚本模板库

**实现建议**：
```swift
// 用户脚本配置
struct UserScript: Codable {
    let id: UUID
    var name: String
    var code: String
    var injectionTime: InjectionTime
    var enabled: Bool
    
    enum InjectionTime {
        case atDocumentStart
        case atDocumentEnd
    }
}

// 在 WKWebViewConfiguration 中注入
let script = WKUserScript(source: scriptCode, 
                          injectionTime: .atDocumentEnd, 
                          forMainFrameOnly: false)
webView.configuration.userContentController.addUserScript(script)
```
